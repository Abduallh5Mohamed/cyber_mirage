1. The Stream Definitions

We define strict channels for data flow to prevent chaos.

class StreamName(Enum):
    """Available message streams"""
    ATTACK_EVENTS = "stream:attacks"      # Raw attacker commands
    AI_DECISIONS = "stream:ai_decisions"  # What the AI decided to do
    FORENSICS = "stream:forensics"        # Evidence collection triggers
    ALERTS = "stream:alerts"              # Critical system warnings

2. The Message Structure

Every message is standardized using a Dataclass to ensure type safety.

@dataclass
class Message:
    message_id: str
    stream: str
    payload: Dict[str, Any]  # The actual data
    priority: int            # 1 (Low) to 4 (Critical)
    timestamp: str
    producer_id: str
    retry_count: int = 0     # For fault tolerance

3. Fault Tolerance (Dead Letter Queue)

If a consumer fails to process a message (e.g., database is down), Role 6 doesn't lose the data. It retries 3 times, then moves it to a "Dead Letter Queue" for manual inspection.

def _move_to_dlq(self, message: Message):
    """Move failed message to dead letter queue"""
    dlq_payload = {
        'original_stream': message.stream,
        'failed_at': datetime.now().isoformat(),
        'reason': 'Max retries exceeded'
    }
    self.redis.xadd('stream:dead_letter', dlq_payload)

üîë Key Capabilities

Feature Description Benefit
Consumer Groups Allows multiple instances of the AI to read from the same stream. Scalability: We can run 10 AI agents in parallel.
Message Persistence Redis saves messages to disk (AOF/RDB). Reliability: No data loss if the server crashes.
Decoupling Honeypots don't know the AI exists. They just talk to Redis. Maintainability: We can upgrade the AI without stopping the Honeypots.
Backpressure Consumers pull data at their own speed. Stability: A flood of attacks won't crash the analysis engine.
